# 技术架构说明

## 1. 项目定位

本项目是一个企业销售数据分析后台，模拟企业销售部门或运营部门日常使用的数据看板系统。系统通过读取销售订单数据，完成指标统计、趋势分析、维度对比、订单明细展示、CSV 数据导入和 AI 经营报告生成。

## 3. 数据流转流程

用户登录系统后，前端会请求后端接口获取销售数据元信息，例如区域、品类、渠道和日期范围。用户应用筛选条件后，前端将筛选参数发送给后端。后端读取 CSV 数据，根据条件筛选订单记录，并计算 KPI、趋势、排行和数据质量结果。最后，后端将分析结果返回给前端，前端再进行图表和表格展示。

## 4. 设计重点

项目重点不是单纯展示图表，而是模拟企业数据分析系统的基本流程：数据进入系统、后端处理数据、前端展示指标、用户根据结果进行业务判断。这个过程能够体现数据分析后台的业务理解和工程实现能力。
